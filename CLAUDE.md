# WC 2026 Predictions Bot — CLAUDE.md

Telegram-бот для турнира прогнозов на Чемпионат Мира 2026.
Разработан для Вани (@ivanshabunin379) и его брата Ника (@Vuexn).

---

## Стек

- **Python** + **aiogram 3.x** (Telegram Bot framework)
- **Supabase** (Postgres, облачный, бесплатный тир)
- **API-Football** (api-football.com) — автоматическое получение результатов матчей
- Деплой: **Railway** или **Render** (бесплатный тир, процесс 24/7)

### Лига 1 — Приватная (Ваня vs Ник)
- Участники: только @ivanshabunin379 (Ваня) и @Vuexn (Ник)
- **Хардкод** — лига создаётся автоматически при старте бота, не через UI

### Лига 2 — Публичная (друзья)
- Открыта для всех по инвайт-ссылке

**Важно:** Ваня и Ник делают ДВА независимых прогноза на каждый матч — один для приватной лиги, один для публичной. Эти прогнозы никак не связаны и могут отличаться.

## Правила приватной лиги (Ваня vs Ник)

### Последовательные прогнозы
На каждый матч заранее определено, кто ставит **первым** (Ваня или Ник):
- Первый делает прогноз → бот уведомляет второго, показывает счёт первого
- Второй делает прогноз → **не может поставить тот же точный счёт**
- После прогноза второго оба видят оба прогноза

### Распределение матчей (кто первый)
Генерируется случайно **до старта турнира** командой `/assignmatches` (только для Вани/Ника).

**Стратификация** по трём уровням для максимальной честности:
1. **По группе** — в каждой группе ровно 3 матча Ване и 3 Нику
2. **По туру** — в каждом туре каждой группы ровно 1 матч каждому (из 2 матчей тура)
3. **По силовому коэффициенту** — внутри каждой пары матчей одного тура группы, "более очевидный" матч (сумма рейтингов FIFA обеих команд) назначается случайно, но с балансом: у Вани и Ника примерно одинаковое количество "предсказуемых" матчей

Итог по групповому этапу: ровно 36/36 матчей.

Плей-офф: распределение решается отдельно перед началом стадии плей-офф (пока не реализовывать, оставить заглушку).

## Система очков

### Групповой этап
| Результат | Очки |
|---|---|
| Точный счёт угадан | 3 |
| Исход угадан (победа/ничья), счёт нет | 1 |
| Мимо | 0 |

### Плей-офф (0–4 очка за матч)

**6 возможных исходов для прогноза:**
- **П1** — победа команды 1 в основное время
- **П2** — победа команды 2 в основное время
- **НП1** — ничья в осн. вр. → команда 1 побеждает в доп. времени
- **НП2** — ничья в осн. вр. → команда 2 побеждает в доп. времени
- **НПП1** — ничья → пенальти → команда 1 побеждает
- **НПП2** — ничья → пенальти → команда 2 побеждает

**Таблица очков** (прогноз → фактический исход):

| Прогноз ╲ Исход | П1 | П2 | НП1 | НП2 | НПП1 | НПП2 |
|---|---|---|---|---|---|---|
| **П1** | **2(4)** | 0 | 1(2) | 0 | 1(2) | 0 |
| **П2** | 0 | **2(4)** | 0 | 1(2) | 0 | 1(2) |
| **НП1** | 1 ⬛ | 0 | **3(4)** | 1 | 2 ⬛ | 1 |
| **НП2** | 0 | 1 ⬛ | 1 | **3(4)** | 1 | 2 ⬛ |
| **НПП1** | 1(1) | 0 | 2(2) | 1 | **3** ⬛ | 2 |
| **НПП2** | 0 | 1(2) | 1 | 2(2) | 2 | **3** ⬛ |

**Формат:** базовые очки **(с точным счётом)**. ⬛ = исключение (см. ниже).

**Примеры:**
- П1 угадан, без точного счёта → 2 очка
- П1 угадан + точный счёт → 4 очка
- НП1 угадан, без точного счёта → 3 очка
- НП1 угадан + точный счёт → 4 очка

**Исключения ⬛** (защита от злоупотребления ставкой на ничью):
1. **НП1 pred + П1 actual** → всегда 1 очко, даже если счёт совпадает
2. **НП2 pred + П2 actual** → всегда 1 очко, даже если счёт совпадает
3. **НП1 pred + НПП1 actual** → 2 очка без бонуса за точный счёт
4. **НП2 pred + НПП2 actual** → 2 очка без бонуса за точный счёт
5. **НПП1 pred + НПП1 actual** → максимум 3 очка (без бонуса за точный счёт)
6. **НПП2 pred + НПП2 actual** → максимум 3 очка (без бонуса за точный счёт)

**"Точный счёт"** в контексте плей-офф = счёт основного времени (для НП/НПП — счёт ничьей в осн. вр.).

## Правила публичной лиги

- Все участники прогнозируют **независимо и слепо**
- Прогнозы других раскрываются только после начала матча
- Те же системы очков (группа + плей-офф)
- Одинаковые прогнозы разрешены
- Нет ограничений на порядок ввода

## UX: флоу прогноза (/predict)

**Умный единый флоу** — пользователь вводит прогноз один раз, выбирает применение.

### Групповой этап
```
Бот: ⚽ Аргентина — Франция | Сегодня 21:00
     Введи счёт (например: 2:1)

Пользователь: 2:1

[Если в одной лиге]:
Бот: Аргентина 2:1 Франция. Верно?
     [✅ Сохранить] [✏️ Изменить]

[Если в двух лигах — Ваня или Ник]:
Бот: Аргентина 2:1 Франция. Применить к:
     [Обеим лигам] [Только личному] [Только общему]
     (если выбрал "Обеим" и в приватной лиге дубль с прогнозом брата — предупреждение)
```

### Плей-офф
```
Бот: Введи счёт основного времени

Пользователь: 1:1

Бот: Ничья — выбери исход:
     [НП1: доп. вр., Аргентина]  [НП2: доп. вр., Франция]
     [НПП1: пен., Аргентина]     [НПП2: пен., Франция]

Пользователь: [НПП1]

Бот: 1:1 → пенальти → Аргентина. Применить к: ...
```

### Приватная лига: порядок прогнозов
```
[Ваня — первый на этот матч]:
Ваня делает прогноз → сохраняется приватно
Бот Нику: "Ваня уже сделал прогноз. Твой ход:"
Ник вводит → если дубль по счёту: "Этот счёт занят, выбери другой"
После прогноза Ника: оба видят оба прогноза
```

## Уведомления

| Когда | Кому | Что |
|---|---|---|
| За 24ч до матча | Всем участникам | Напоминание о матче + кнопка [Сделать прогноз] |
| За 2ч до матча | Кто ещё не сделал прогноз | Последнее напоминание |
| В момент старта матча | — | Приём прогнозов закрыт |
| После финального свистка (~2ч после начала) | Всем участникам | Результат + набранные очки + текущий счёт в турнире |

---

## Дедлайн и редактирование

- Прогнозы принимаются до момента начала матча (kickoff)
- После подтверждения (`[✅ Сохранить]`) прогноз изменить нельзя
- Перед сохранением всегда показывается экран подтверждения

---

## Автоматические результаты

Бот сам получает результаты матчей через **API-Football**:
- Через ~2 часа после начала матча запрашивает результат
- Рассчитывает очки для всех прогнозов
- Рассылает итоги участникам

Ручной ввод `/result` не нужен (только как крайний fallback при недоступности API).

## Схема базы данных (Supabase/Postgres)

```sql
users
  id, telegram_id, username, name, created_at

leagues
  id, name, type ('private' | 'public'), created_at

league_members
  league_id, user_id, joined_at

matches
  id, home_team, away_team, kickoff_at, stage,
  group_name, round, home_score, away_score,
  outcome ('P1'|'P2'|'NP1'|'NP2'|'NPP1'|'NPP2'), status

match_assignments
  match_id, first_user_id   -- кто ставит первым (только приватная лига)

predictions
  id, user_id, match_id, league_id,
  home_score, away_score, outcome_type,
  points, created_at

invite_tokens
  token, league_id, created_by, expires_at, used
```
